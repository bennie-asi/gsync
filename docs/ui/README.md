# GSYNC UI Prototype Docs

This folder documents the Stitch prototype set for the GSYNC desktop application.

## Files
- `prototype-inventory.md` — full screen inventory, Stitch screen IDs, refined variants, state variants, and information architecture
- `implementation-notes.md` — recommended component patterns, implementation order, and UX guidance

## Preferred prototype set
Unless otherwise noted, implementation should follow the **refined** screens rather than the first-pass screens.
For empty states and diagnostic/error cases, use the dedicated **state variant** screens as the reference.
