## Why

当前主界面与 Stitch refined Library 设计稿仍有多处偏差，已经影响到页面信息架构、功能区布局与状态表达的一致性。现在需要把主界面重新收敛到 Stitch 设计稿，避免后续继续在错误布局上叠加实现。

## What Changes

- 将主界面壳层重新对齐到 Stitch refined Library 设计稿，确保左侧为固定菜单区，顶部展示头像、应用名与版本信息。
- 调整 Library 主功能区结构：顶部左侧为占主功能区约一半宽度的搜索输入框，右侧为排序与刷新按钮。
- 调整 Library 内容布局：中部为游戏表格区域，右上方为“添加游戏”和“立即同步”主操作按钮，右侧下方为总览视图。
- 固定页面底部状态栏，左侧展示 WebDAV 在线状态与 Local 就绪状态，右侧展示 WebDAV 名称与同步状态。
- 统一表格状态列的多语种彩色语义，确保中文状态文本与英文状态文本一样保留对应的颜色语义，而不是退化为单一颜色。
- 收紧实现约束：Library 页面必须以 Stitch 设计稿为准，不允许擅自改变信息层级、布局分区或状态表达。

## Capabilities

### New Capabilities
- `localized-status-badge-semantics`: 定义表格与状态区在多语言显示下仍保持一致颜色语义的状态标记规则。

### Modified Capabilities
- `stitch-app-shell`: 调整应用壳在 Library 页面中的左侧菜单头部与底部固定状态栏要求。
- `library-and-details-ui`: 调整 Library 页面主功能区、表格区、右侧操作区与总览区，使其对齐 Stitch refined 设计稿。

## Impact

- 影响现有主界面布局实现与共享壳层组件约束。
- 影响 Library 页面中的工具栏、表格、右侧概览区与底部状态栏实现。
- 影响状态徽标/状态文本在本地化场景下的颜色映射与展示逻辑。
- 影响后续 UI 实现验收标准，需以 Stitch 项目 `13407775155513183369` 与设计系统 `assets/3772949807639653402` 为主要参考。
